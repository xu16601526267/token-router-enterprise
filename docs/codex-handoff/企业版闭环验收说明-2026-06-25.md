# Token Router 企业版闭环验收说明（2026-06-25）

## 范围边界

- 本轮按“除 Go 版本暂不调整外，其余企业版剩余清单尽量闭合”的口径处理。
- Go 版本不在本轮改动内；当前验证使用本机现有 Go 工具链。
- 本轮只更新本地 git，不推送远端。

## 下游客户主链路

1. 管理员在企业 API Key 页面为下游客户或团队创建 Key，可限制额度、模型、路由分组、IP 白名单、跨组重试和有效期。
2. 下游客户用 Key 调 OpenAI-compatible API 后，系统落 `logs` 与 `usage_ledgers`，企业用量页按关键词、模型、状态、渠道、用户和分组聚合。
3. 下游客户或二级中转站可用同一 API Key 查询 `/v1/dashboard/billing/usage`、`/v1/dashboard/billing/subscription` 和 `/api/log/token` 做 Key 维度对账；这些只读对账入口不再受固定 IP 限流影响。
4. 真实企业客户场景下，A 平台给多个 B 企业客户分别开上游 Key；每个 B 可在自己的站内给多个 C 员工/下游用户创建多个 API Key；C 的请求先在 B 侧按 C-Key 记账，再由 B 通过 A 侧 B-Key 转发并在 A 侧记账。
5. 运营可在渠道与供应商中心查看上游渠道健康、成功率、延迟、余额、供应商画像和近期异常。
6. 财务可在计费与结算中心直接生成客户或供应商结算单，并导出账单、渠道、API Key、用量明细 CSV 做对账。
7. 管理员与普通管理员的数据范围按角色收敛；普通管理员不能管理 root 或同级管理员资源。

## 新增或补齐的企业接口

| 模块 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 渠道中心 | GET | `/api/enterprise/channels` | 支持 `keyword/status/supplier_id/group/page/page_size/sort_by/sort_order` 服务端筛选分页 |
| 渠道中心 | GET | `/api/enterprise/channels/export` | 按当前筛选导出渠道与供应商 CSV，最多 10000 条 |
| 渠道详情 | GET | `/api/enterprise/channels/:id` | 渠道画像、供应商画像、支持模型、近期错误事件 |
| 用量分析 | GET | `/api/enterprise/usage-analytics` | 支持关键词、模型、用户、分组、状态、渠道、分页和排序 |
| 用量导出 | GET | `/api/enterprise/usage-analytics/export` | 按当前筛选导出指标、趋势、成本分布、调用日志 CSV |
| 结算中心 | POST | `/api/enterprise/billing/settlements/generate` | 从企业结算页生成客户或供应商结算单 |
| 账单导出 | GET | `/api/enterprise/billing/export` | 导出账单指标、收支趋势、结算单和充值流水 |

## 权限与数据隔离

- `/api/enterprise/*` 路由统一挂在 `AdminAuth()` 下。
- 企业 API Key 查询、导出、创建、编辑、轮换、删除使用 `enterpriseCanManageUser` 检查管理范围。
- root 可管理全部用户；普通管理员只能管理自己和低角色用户，不能越权查看 root 或同级管理员 Key。
- 导出的 API Key 使用脱敏字段，不输出完整密钥明文。
- 企业用户列表复用同一管理范围，避免普通管理员看到 root 或同级管理员的用户与密钥统计。
- 下游只读对账入口仍必须通过 API Key 鉴权；本轮只移除固定 IP 限流，不开放匿名访问，不改变额度、模型白名单和 Key 状态校验。

## 数据库与性能口径

- `group` 作为保留字时使用项目已有方言列名工具，兼容 SQLite、MySQL、PostgreSQL。
- 企业用量和渠道中心的分页查询拆分为独立 `COUNT` 与 `Find`，避免 GORM 查询对象复用导致分页结果被统计查询污染。
- 用量日志聚合统一使用 `channel_id` 字段；渠道详情近期异常也使用同一字段。
- CSV 导出限制为最多 10000 条，避免大表一次性导出拖垮管理页面。
- 页面侧服务端筛选替代本地筛选，数据量增大后仍能保持总量、导出和页面口径一致。

## 前端闭环

- 渠道与供应商中心：服务端筛选、分页、导出渠道、详情侧栏、供应商画像、近期异常。
- 用量日志与成本分析：服务端筛选、分页、总量显示、筛选后导出、模型/状态/关键词一致口径。
- 计费与结算中心：企业页内生成结算单、导出账单、单张结算单下载、毛利趋势字段修复。
- 企业 API Key：前序已完成创建、编辑、轮换、删除、导出和权限范围校验。
- 控制塔、用户治理、个人工作台保留前序已完成的企业 UI 与真实数据入口。
- 前端全仓质量门禁已跑通：类型检查、生产构建、lint、格式检查和版权头检查均通过；lint 仍保留非阻断历史警告。
