# 企业版 UI 补全与复测报告（2026-06-25）

## 本次完成范围

- 全局外壳：顶部栏补充企业工作区、环境和日期信息，侧边栏改为更稳定的企业深色导航风格。
- 企业页头：去掉装饰性背景块，改为更克制的卡片式信息区，和原型图的企业后台风格保持一致。
- 企业总览：修正用量日志入口路由类型，避免跳转到不存在的 section。
- 控制塔：`新建路由策略` 不再是空按钮，点击后进入 Routing Policies 工作区，并提示从自托管执行来源激活策略。
- 用户治理：接口补齐 `quota` / `request_count` 字段，右侧成员详情可正确显示可用额度和累计请求；`在经典管理中编辑` 会滚动到真实经典用户管理表格。
- 计费与结算：补齐 `生成结算单`、`导出账单`、`费用分析`、单张结算单下载等操作；账单导出为 UTF-8 CSV。
- Classic 构建：为 classic 前端限定 Semi UI 旧链路使用 `date-fns@2.30.0`，避免构建时解析到 workspace 顶层 `date-fns@4`。
- 构建产物：`web/default/dist` 与 `web/classic/dist` 属于构建产物，不纳入源码提交，已加入 ignore。

## 本轮补全闭环

- 企业 API Key：新增后端 CSV 导出接口，导出当前管理员权限范围内的脱敏清单，不返回完整 Key；前端列表页新增导出按钮。
- 用量与成本分析：趋势接口补齐 `prompt_tokens`、`completion_tokens`、`average_latency_ms`，日志状态统一为 `success/error`；新增服务端关键词、模型、用户、分组、状态、渠道、分页和排序过滤。
- 用量导出：新增后端 CSV，包含指标、每日趋势、模型/用户/渠道/分组成本分布、最多 10000 条调用日志，并严格复用当前筛选条件。
- 渠道与供应商中心：新增服务端关键词、状态、供应商、分组、分页和排序过滤；新增渠道中心 CSV 导出，避免大客户数据量下前端本地筛选口径失真。
- 计费与结算：导出账单改为后端生成，包含账单指标、收支趋势、最多 10000 条结算单和充值流水；企业结算页可直接按用户或供应商生成结算单，前端保留单张结算单下载。
- 控制塔：风险提醒可确认/忽略，待执行动作可开始/标记完成，活跃策略可停用，并同步刷新企业控制塔和原 Token Router 页签缓存。
- SQL 兼容性：修复 `group` 保留字在 SQLite/PostgreSQL/MySQL 下的列名转义；企业用量、用户分组、密钥搜索统一使用方言安全列名。
- 日志渠道聚合：企业控制塔、渠道中心、用量分析统一使用 `logs.channel_id`，避免新库/测试库查旧 `channel` 列失败。
- 权限范围：企业用户列表和企业 API Key 查询/导出均按管理员角色范围收敛，普通管理员不能看到 root 或同级管理员资源。
- 测试隔离：新增企业测试使用包级共享测试库并清理相关表，不再关闭全局 DB 影响后续测试。
- Go 质量门禁：清理 `go vet` 拦截项，去掉 SSE `CustomEvent` 中无效的值复制 mutex，并删除多个渠道适配器中 `panic` 后不可达 return。
- 交付文档：新增 `企业版闭环验收说明-2026-06-25.md`，按下游客户主链路、接口、权限、数据库、前端和验收用例整理。
- 前端全仓质量门禁：补齐版权头、格式化全仓、收敛阻断级 lint 规则，修复类型安全和 iframe sandbox 等阻断项。
- 真实下游验收：新增 `deploy/smoke/enterprise-downstream-customer-smoke.sh`，用真实 API Key 覆盖余额、用量、单次、长输出、并发、错误模型和 Key 日志。

## 已知非本次遗留

- Go 版本按用户要求暂不调整；当前验证使用本机现有 Go 工具链。
- 前端 `npm run lint` 当前为通过状态，但仍输出若干非阻断历史警告，主要是嵌套三元表达式、数组 index key、循环依赖、`dangerouslySetInnerHTML` 等；这些不再阻断构建，本轮未做大规模语义重构。
- 真实下游 smoke 已覆盖 API Key 视角；浏览器端 Playwright 自动化尚未引入，后续如要覆盖页面点击流，可在该 smoke 基础上追加 Playwright 用例。
