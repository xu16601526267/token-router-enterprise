# 下游客户真实环境 Smoke（2026-06-25）

## 脚本位置

- `deploy/smoke/enterprise-downstream-customer-smoke.sh`
- `deploy/smoke/enterprise-downstream-live-matrix.py`
- `deploy/smoke/enterprise-b-reseller-c-chain.py`

`enterprise-downstream-customer-smoke.sh` 只使用下游 API Key，不需要管理员权限，适合快速模拟：

- 直接使用者调用中转站。
- 另一个中转站把本站作为上游。
- B/C 端客户拿 Key 后做余额、调用、错误模型和日志核对。

`enterprise-downstream-live-matrix.py` 使用测试账号登录后自动创建临时 Key，覆盖 C 端直连、B 端二级中转、并发、流式、负向 Key 和对账链路；脚本结束会删除临时 Key，报告只保留脱敏 Key。

`enterprise-b-reseller-c-chain.py` 覆盖真实 A/B/C 链路：A 是当前平台，B 是临时启动的企业客户中转站，C 是 B 站内创建的员工/下游用户。脚本会给每个 B 在 A 侧创建独立上游 Key，再在每个 B 里创建多个 C 用户和多个 C API Key，并发打 C->B->A 请求后分别核对 A 侧 B-Key 账本和 B 侧 C-Key 账本。

## 用法

```bash
TOKEN_ROUTER_BASE_URL='https://approaching.aimaserver.com' \
TOKEN_ROUTER_API_KEY='替换为下游 API Key' \
TOKEN_ROUTER_MODEL='kimi-test' \
TOKEN_ROUTER_CONCURRENCY=6 \
TOKEN_ROUTER_MAX_TOKENS=128 \
TOKEN_ROUTER_LONG_MAX_TOKENS=512 \
deploy/smoke/enterprise-downstream-customer-smoke.sh
```

正式账本验收可以加严格模式：

```bash
TOKEN_ROUTER_STRICT_LEDGER=1 deploy/smoke/enterprise-downstream-customer-smoke.sh
```

严格模式下，如果后置用量没有增长或 Key 日志少于本轮请求数，脚本会返回失败。

## 覆盖维度

- `/v1/dashboard/billing/subscription`：Key 维度余额查询。
- `/v1/dashboard/billing/usage`：Key 维度用量查询。
- `/v1/chat/completions`：单次 OpenAI 兼容请求。
- `/v1/chat/completions`：长输出请求。
- `/v1/chat/completions`：多并发请求。
- `/v1/chat/completions`：错误模型应被拦截。
- `/api/log/token`：Key 维度日志查询。
- 自动输出每个请求的 HTTP 状态、耗时、响应体和 `summary.md`。
- C 端直连：单模型白名单、非流式、中文、流式、并发、错误模型、余额/用量/日志对账。
- B 端二级中转：多模型发现、主模型/次模型、流式、并发、错误模型、余额/用量/日志对账。
- 真实 A/B/C 链路：A 下挂多个 B 企业，每个 B 下挂多个 C 员工，每个 C 多个 API Key；C 并发请求 B，B 再转发到 A。
- 多租户账本：B 侧逐个 C-Key 查询用量和日志；A 侧逐个 B 上游 Key 查询用量和日志；非法模型由 B 拦截且不进入 A 侧账本。
- 拒绝链路：低额度 Key、禁用 Key、错误 Key、缺失 Key。

## 本轮发现并修复的问题

- 问题：20 并发真实矩阵中，业务请求全部成功，但 `/api/log/token` 在高频对账时返回 `429`，导致下游客户无法稳定拉取 Key 日志做账务核对。
- 根因：`/api/log/token` 之前挂在 `CriticalRateLimit` 下，同时 `/api` 全局限流按来源 IP 计数；B 端二级中转会把大量客户请求聚合到同一个出口 IP，不能用固定 IP 限流挡对账接口。
- 修复：`/api/log/token` 和 `/api/usage/token/` 保留 `TokenAuthReadOnly` 鉴权，但移出关键操作限流；`GlobalAPIRateLimit` 对这两个下游只读对账入口豁免。登录、支付、密钥明文查看、后台搜索等敏感入口仍保留限流。
- 线上：2026-06-25 18:01 CST 已在 `token-router.service` 部署重启，替换前已备份线上二进制。
